# PM Self-Check：产品自查 & 产品自问

帮助产品经理在行动前确认：自己是不是正在朝一个更对的方向走。
**不替用户下结论，帮用户识别假设、暴露盲区、明确下一步验证点。**

## 执行流程

### 第一步：收集最小必要信息

进入框架前先了解：
- 产品是什么、处于什么阶段（新产品 0→1 / 已上线优化 / 老产品复盘）
- 目标用户是谁，解决什么核心问题
- 当前最大的担心 / 分歧 / 症状是什么
- 已有哪些证据（用户访谈、数据、市场反馈）

信息不足时，先补齐关键上下文，再进入判断。

### 第二步：产品自查

读取 `references/framework.md` 中的「产品自查」部分，从三个维度评估当前状态：
1. 产品方向和产品体验
2. 迭代流程和问题呈现
3. 核心价值和功能交互（AI 时代）

### 第三步：产品自问（8 问）

读取 `references/framework.md` 中的「产品自问」部分，引导用户过完 8 个问题：
- **新产品**：先聚焦上篇「社会竞争」4 问
- **老产品**：结合自查结论，聚焦最相关的问题，允许裁剪

## 使用原则

- **先诊断，再价值校准**：先判断「能不能成立」，再判断「值不值得做」
- **证据优先**：追问用户证据（行为数据、访谈、留存/转化/复购），不足则标注「待验证」
- **允许裁剪**：B 端工具、基础设施、强运营驱动的场景，可跳过不适用的问题，不强行走完
- **不替用户拍板**：结论不确定时，如实说「目前无法判断」

## 输出结构（灵活使用，不必逐项机械填写）

1. 背景复述 + 阶段判断
2. 产品自查结论（三维度：信号 / 风险点 / 是否需更多证据）
3. 8 问摘要（可用标签：成立 / 风险高 / 暂不成立 / 待验证）
4. 当前最关键的盲区
5. 下一步 1～3 个验证动作（优先级从高到低）
